## Что программа умеет сейчас

### 1. Принимает входные файлы (PDF + Lua)
- Читает PDF-документы через Apache PDFBox
- Читает Lua-скрипты
- Извлекает базовую информацию

### 2. Строит доменную модель проекта
- Создаёт модель оборудования (`Equipment`)
- Создаёт модель тегов/сигналов (`Tag`)
- Создаёт модель логики управления (`ControlLogic`)

### 3. Генерирует XML-проект
- Создаёт файл `project.xml` с оборудованием, тегами и Lua-кодом
- Структура XML соответствует доменной модели

### 4. Имеет CLI-интерфейс
- Команда `generate` — полная генерация проекта
- Команда `update` — заготовка для инкрементального обновления
- Команда `watch` — заготовка для мониторинга изменений

## Ответы на возможные вопросы

### "Насколько это готово к использованию?"

**Ответ:**
> "Сейчас это рабочий MVP. Программа генерирует XML из PDF и Lua, но в базовом формате. Для реального использования нужно доработать генератор под формат Monitor и улучшить парсеры. Оценка: ~30% готовности к продакшену."

---

### "Когда будет готово?"

**Ответ:**
> "План: следующий этап (генератор Monitor) — 2-3 недели. Полная версия с инкрементальным обновлением — 1-2 месяца. Зависит от сложности парсинга PDF и формата Monitor."

---

### "Можно ли использовать для других проектов?"

**Ответ:**
> "Архитектура позволяет легко добавлять новые парсеры и генераторы. Но сейчас мы фокусируемся на проекте 'Моцарелла' как эталоне. После его реализации можно расширить на другие проекты."

---

### "Как тестируется?"

**Ответ:**
> "Есть unit-тесты для ключевых компонентов. Планируем интеграционные тесты, которые сравнивают результат генерации с эталонными файлами Monitor из проекта 'Моцарелла'."

---

### "Что с инкрементальным обновлением?"

**Ответ:**
> "Интерфейс уже есть (`DeltaUpdater`), но реализация — заглушка. Это следующая большая задача после генератора Monitor. Нужно научиться сравнивать модели и обновлять только изменения."
