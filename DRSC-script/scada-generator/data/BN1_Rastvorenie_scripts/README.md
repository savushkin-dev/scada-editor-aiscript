# Скрипты BN1 (Monitor)

## Что лежит в папке

- **`scripts.txt`** — выгрузка/сборник **скриптов визуализации Monitor** для проекта BN1 «Растворение» (синтаксис как в `Units.script`: `function extern`, `endf`).

Это **не** управляющий Lua контроллера из `BN1_Rastvorenie/docs/*.lua` — другой слой: обработчики отражений на картах и вызовы библиотечных функций **`PSF.*`**, **`prj.*`**.

## Что внутри по смыслу

- **`OnClickR_*`** — клик по отражению; часто парсят **`selfr.rname`** (имя отражения) подстроками `reflname[i,len]`, собирают массив **`Message`** и вызывают **`PSF.EditNumTankR`**, **`PSF.SetLineInParamsRegim`**, **`PSF.UprMaster`** и т.д.
- **`OnGetColorR_*`** — цвет по состоянию, типично **`return( prj.GetIOColor( ... ) );`**.

В файле порядка **сотен** объявлений `function extern` (в основном **OnClick** и **OnGetColor**; отдельных **`OnChange*`** в этом тексте может не быть — для BN1 логика смены свойств может быть в других местах выгрузки).

## Связанные файлы

- **`type-mapping.yaml`** — правила: новый тег Lua (префикс `L##…`) → шаблоны `OnClickR_L{line}_P##` из этого каталога.

## Зачем это `scada-generator`

1. **Эталон реальных имён функций** и тел для генерации/мержа **`Units.script`** (следующий этап после `Units.xml`).
2. **Словарь вызовов `PSF.*`** — можно автоматически построить список используемых API для документации или проверок.
3. **Связка имени отражения ↔ обработчик** — основа для будущего маппинга «элемент карты → блок в `Units.script`».

## Ограничение

Корректность в конкретной версии Monitor нужно подтверждать с инженерами АСУТП; для автотестов генератора достаточно фиксировать **небольшие выдержки** из этого файла как golden-сниппеты.

## Уникальные вызовы `PSF.*` в `scripts.txt` (по тексту файла)

`PSF.EditNumTankR1`, `PSF.EditNumTankR`, `PSF.EditNumTankRA`, `PSF.SetLineInParamsRegim`, `PSF.SetLOParamsRegim`, `PSF.UprMaster`, `PSF.EditNumTIn`, `PSF.EditNumTOut`, `PSF.EditNumTankV`, `PSF.EditNumTank`, `PSF.EditNumTank_Out`, `PSF.SetTankRegim`, `PSF.SetMixRegim`, `PSF.SetCoolRegim`.

Примечание: попытка собрать этот список одной строкой `powershell -Command "… $_.Matches …"` без экранирования `$_` даёт ошибку; для повторения анализа удобнее поиск по репозиторию или скрипт в отдельном `.ps1` файле.
