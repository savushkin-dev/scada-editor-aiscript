## Синтаксис 

```Исходный текст = <TEXT>

TEXT = <Блок выражений>
<Выражение>
Блок выражений = <Выражение>1;[...<Выражение>N;]
Выражение = [#nn:] <Оператор управления>
<Арифметико-логическое выражение>
```

## Операторы управления

### Не выполняемые операторы

1. Определение функции NameFunction. Если указано ключевое слово

``` 
Function [Extern] NameFunction [<список аргументов>]
<TEXT>
EndF
```

**Extern** - функция может быть вызвана из DELPHI или из других
модулей. Список аргументов определяет соответствие аргументов
функции параметрам вызова. 

Если непосредственно перед EndF нет
оператора **RETURN** - автоматически вставляется **Return(NULL)**.

2. ```Static <список переменных>``` - объявление статических переменных.
Оператор может находиться в любом месте модуля. Переменная известна
всюду ниже определения.

Переменная Static существует вместе с модулем (создается при трансляции).

Доступ к переменной из других модулей : ```<модуль/блок>.<переменная>```.

3. **Local** ```<список переменных>``` - объявление локальных переменных.
Оператор может находиться в любом месте модуля внутри функции.
Переменная известна всюду ниже определения внутри функции.
Переменная **Local** создается при вызове функции и уничтожается
после завершения ее работы.

4. ```Const C1=<Константа>[,C2=<Константа>...[,Cn=<Константа>]]]```

### Выполняемые операторы.

```
0. TRY
<TEXT1>
EXCEPT | FINALLY
<TEXT2>
END;
```

Блок обработки ошибок. Если в процессе выполнения **<TEXT1>** произошла ошибка, управление немедленно передается на **<TEXT2>**, послевыполнения которого для **Except** выполнение продолжается после **END**,для **FINALLY** - повторно генерируется ошибка. Если в **<TEXT1>** ошибки не было, то для **Except <TEXT2>** пропускается, а для **Finally** - выполняется.

1. **Public <список переменных>** - определение глобальных переменных.
После выполнения оператора переменная может быть использована всюду (в этом экземпляре класса ```RunExpr```). !!!. PUBLIC-переменные намного медленнее всех других !!!
По возможности необходимо избегать их использования в циклах или использовать доступ по адресу.

2. **Release <список переменных>** - удаление глобальных переменных.

3. Условный оператор. **<TEXT1>** будет выполнен если значение выражения = True. Иначе выполняется **<TEXT2>**
```
IF (<лог. выражение>)
<TEXT1>
[ELSE
<TEXT2>]
END
```

4. Оператор цикла. **<TEXT>** будет выполняться пока значение выражения = **True**.

```
While (<лог. выражение>)
<TEXT>
END
```

Оператор цикла. **<TEXT>** будет выполняться пока значение выражения = **True**.

5. Оператор цикла. **<TEXT>** выполнится 1 раз и затем будет выполняться пока значение выражения = **False**.

``` Repeat
<TEXT>
Until (<лог. выражение>)
```

6. **Break** - Немедленный выход из цикла.
7. **Goto(nn)** - Переход на метку nn;
8. **Return(<Выражение>)** - Возврат из функции.
9. **Continue** - Следующая итерация цикла.

## Арифметико-логическое выражение

Арифметико-логическое выражение может включать ОПЕРАНДЫ, ОПЕРАЦИИ и круглые скобки для изменения последовательности вычислений.

ОПЕРАНДЫ :

1. Константа (Число,Строка,True,False,Null)
2. Идентификатор

При трансляции модуля происходит немедленное связывание иденитификаторов с переменными в следующей последовательности :

* Local
* Static

Для неразрешенных ссылок при выполнении происходит поиск в Public-переменных. Если переменная не найдена - производится обращение к полю базы с указанным именем. Для исключени неоднозначности можно использовать ```FIELD-><имя поля базы данных>```.

3. Вызов функции[ (список аргументов) ]

4. Функция управления

```
IIF(<Лог.выражение>;<выражение1>;<выражение2>)
```

## Массивы

### Создание массива 

1.Создать двумерный массив (2 строки, 3 столбца) все значения равны Null.

```
Ar:=Array(2,3)
```

2. 

```
Ar:=[[Null,Null,Null],[Null,Null,Null]]   -- // --
```

3. 

```
Ar:=[1,2,3,[41,42,43],n,m]
```

Любой элемент массива сам может быть массивом. Допустимы конструкции :

```
Ar[n]:=Array(10) или Ar[n]:=[m1,m2,m3,[m41,42]]
```

Массив может быть передан в другие функции и возвращен ими в качестве результата. При присваивании массива другой переменной копирование элементов не происходит. Массив доступен из двух переменных до присваивания в них других значений и существует, пока на него ссылается хотя бы одна переменная.

**Важно:** 

```
m1:=[1,2,[31,32,33]];
m2:=m1;
m2[1]:='9';
m1:=asize(m1,5);
m2[0]:=0;
m2[2,1]:=99;
```

После выполнения фрагмента 

```
m1=[1,9,[31,99,33],Null,Null], m2=[0,9,[31,99,33]].
```

Для работы с массивами в DELPHI предназначены функции **Clip**  >(см.P_Utils.pas)    

Обращение к элементам массива :  V[I1...[,In]] ;

* к свойству объекта по умолчанию :  O[I1...[,In]] ;
* к свойству-массиву              :  O.Prop[I1...[,In]] ;
* через указатель                 :  P[i] ;
 *к подстроке                     :  S[n,[l]] ;
* кроме того при присваивании  :  S[n,Null] или S[n,];

Пример : 

```
S:='1234567890';
s1:=s[3,2]     ;  // s1='34'
s1:=s[3]       ;  // s1='34567890'
s[3,2]:='ABC'  ;  // s ='12ABC567890'
s[3]:='ABC'    ;  // s ='12ABC'
s[3,]:='ABC'   ;  // s ='12ABC67890'
```

### Типизированые массивы.

Создание массива :

```
ArrayType(n,vType) - создает массив n элементов указанного типа.
```

Допустимые типы  :

* VarByte;
* VarSmallInt;
* VarInteger;
* VarString;
* VarBoolean;
* VarSingle;
* VarDouble;
* VarCurrency.

Типизированные массивы могут иметь только одно измерение. В остальном работа с ними идентична работе с обычными массивами. Выражение **@m[i]** для такого массива возвращает типизированный указатель.

### Массивы Delphi.

Создание массива :

```
VarArrayCreate(m,vType)
``` 

Cоздает вариантный массив Delphi указанного типа. m-массив границ измерений массива.

Пример : 

```
VarArrayCreate([2,8,5,6],varInteger)
``` 

создает массив с двумя измерениями с границами от 2 до 8 в первом и от 5 до 6 во втором.

Получить адрес элемента массива Delphi невозможно. Функция **VarArrayLock** возвращает адрес 1 элемента массива.

Методы:

```
PrintReport(kRec[,pZap=True[,mVars]])
```

Получить отчет с кодом kRec.

* pZap  - признак выдачи запроса.
* mVars - массив значений переменных, например [['VCOPIES',3],['VSKLAD','0100000000]]

```OpenTables(Nmb1[,..Nmbn])``` - открыть таблицы, заданные параметрами. Возвращает массив объектов TTable.


```Choice(Nmb,Sp,[,Form='Formbd1'[,Sd=''[,Mult=False]]]``` - Выбор из базы данны с именем Nmb списка полей Sp на форме Form c нач.поз. Sd. Mult – признак множественного выбора. Возвращает массив полей (Mult=False) или двумерный массив записей-полей (Mult=True).

```Getzap(kZap)``` - возвращает запрос с кодом kZap или генерирет ошибку.

## Класс TInterVars

Класс **TInterVars** используется для создания объектов содержащих свойства с произвольными именами.

Метод **o.Add(Name)** добавляет объекту o свойство с именем **Name.o.Del(Name)** удаляет свойство.

Доступ к свойствам класса достаточно медленен и использование класса оправдано при необходимости облегчения читаемости текста.
Пример :

```
Line:=CreateObj('TInterVars');
Line.Add('X1','Y1','X2','Y2');
Line.x1:=10;
Line.y1:=10;
Line.x2:=100;
Line.y2:=100;
```

## Класс TInterDll

Класс **TInterDll** обеспечивает вызов процедур и функций из Dll-библиотек, а также вызов по абсолютным адресам.

Свойства: 

* Name    : String - Имя библиотеки.
* TypeCall: Byte   - Тип вызовов подпрограмм(Register,Pascal,Cdecl,StdCall,SafeCall).

Методы :

* LoadDll(Name) - загружает библиотеку с именем Dll => Boolean.
* FreeDll       - освобождает библиотеку.
* CallProc(Id[,Par1...Parn]) - Вызывает процедуру с именем(номером) или адресом ID с передачей параметров Par1-Parn.
* CallFunc(Id,Shab[,Par1...Parn]) - Вызывает процедуру с именем(номером) или адресом ID с передачей параметров Par1-Parn. Shab - значение, имеющее тип возвращаемого результата функции.

Пример :

```
      Dll:=CreateObj('TInterDll');
      Dll.LoadDll('DllTest');
      // В Dll функция FunTest описана :
      //   Function FunTest(var s:string;var i:Integer;v:variant;
      //                        a:array of Byte):^Double;
      s:=VarCast('Test',varString);
      i:=VarCast(77,varInteger);
      a:=ArrayType(12,varByte);a[6]:=66;
      Id:='FunTest';
// Для вызова по порядковому номеру :
//    Id:=VarCast(Num,VarSmallInt);
      u:=Dll.CallFunc(Id,varcast(@i,varDouble),@s,@i,
                         Varcast(12.76,varVariant),a);
      qout(u^,s,i); // Результат и Var s,i
      Dll.FreeDll;
      FreeObj(Dll);
```

Для вызова не DLL програм по адресам нужно создать объект DLL(или использовать существующий), установить тип вызова и передать Id типа **Integer** или **Pointer**.
