# Заготовки обработчиков (onchange / onclick)

Файлы в этой папке — **не продакшен**, а **ориентир для следующей итерации** генератора и для объяснения на демо.

## Откуда взята форма

В реальном проекте Monitor обработчики для юнитов часто живут в **`Units.script`**, а не только в `Units.xml`.

Структура блока и синтаксис (`function extern`, `endf`) согласованы с эталоном:

`data/MOZARELLA/O1_MOZARELLA.files/O1_MOZARELLA.files/Units.script`

(см. блоки `//--------------- Begin: ИМЯ_ЮНИТА ---------------`).

В **`Maps.xml`** у отражений (`reflection`) задаются уровни прав для событий в узле `<events>` (например `onclickpermissionslevel`).

## Файлы

| Файл | Назначение |
|------|------------|
| `Units.script.stub.example` | Минимальный вымышленный юнит с `OnChange*` и `OnClick*` — образец целевого выхода генератора для `Units.script`. |
| `maps-events-snippet.example.xml` | Усечённый фрагмент структуры карты Monitor (узел `<events>` у отражения). |

## Ограничение

Перед использованием в рабочем проекте нужна **проверка инженерами АСУТП** (версия Monitor, допустимые вызовы API скриптов).
